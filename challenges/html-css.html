Core HTML and CSS concepts

Summary

Every discipline starts somewhere, and client-side web development starts with HTML (Hyper-Text Markup Language) and CSS (Cascading Style Sheets). HTML is a descriptive markup language that enables us to add meaning to otherwise plain text. CSS is a declarative language that enables us to define formatting rules for our HTML. When combined together, these two languages allow us to write semantically-rich and visually-appealing pages.

During this challenge, you will get a chance to practice and demonstrate your proficiency in basic HTML and CSS. You will build a simple but attractive single-page web site for a fictitious restaurant. You will use git and GitHub to manage your project, and deploy the site to your student web server.

Objectives

By completing this challenge, you will practice and demonstrate the following skills:

  • Writing semantically-rich HTML pages by choosing the most appropriate elements to wrap your information
  • Writing well-formatted HTML source that is properly-indented and easy to read
  • Creating hyperlinks that tie pages together
  • Using HTML character entities to represent reserved or difficult to input characters
  • Building relative URLs that refer to other resources in your project
  • Creating external stylesheets and linking them to your web pages
  • Using a normalizing stylesheet to normalize the various default formatting provided by different web browsers, giving you a consistent foundation upon which to build
  • Writing style rules and using styling properties to control the formatting of your pages, including fonts, colors, links, backgrounds, borders, padding, and margins
  • Using various CSS units, including percentages, pixels, ems, hex colors, and rgba() colors

Menu Section

After the general info section, add another section for the menu, which is this part of the page:

Use the most appropriate elements for the second-level ('Our Menu') and third-level headers ('Meat Pies' and 'Veggie Pies').

The most appropriate elements to use for the menu items is a tricky question. Your first instinct might be to just use normal paragraphs, but that doesn't encode any real meaning about that content. A menu is a list of things, so we should at least use some kind of list element. Your second thought might be to use a simple unordered list (<ul>), but that element is for a list of items where each item is independent of all the other items. Look at the menu again. It's actually comprised of a set of name and description pairs. Use the elements that are appropriate for a list of descriptions.

Note that the name of the first pizza has an ampersand in it. The ampersand character is a reserved control character in HTML, so you have to use the equivalent HTML character entity for the ampersand.

For the pizza descriptions, just use a run of 'lorem ipsum' text. This sort of text is used as a placeholder so that page designers can see what the page will look like before the final text has been written. If you are using a text editor that supports it, just type lorem and hit tab to generate a run of lorem text. If your editor doesn't support that, you can use a web-based lorem generator. Generate a short paragraph of text; the exact number of words is not important.

Add the Style Rules

Your HTML page should now be complete, and if you open it in a web browser, you should see a fairly plain-looking version of the page. To make it look more like the screen shots, we need to add some styling rules.

Open the css/main.css file in your text editor. This is where we will specify all our styles for the page.

First, we need to import the two fonts used by this page: Raleway (for the headings) and Lato (for the rest of the text). Just as we did in class, use Google Fonts to generate @import declarations that include the 300 weight of Raleway, and the 300 and 400 weight of Lato. Add those @import directives to the top of your css/main.css file.

The Google Fonts tool will generate URLs in the @import directives with the http:// protocol, but you might want to edit those directives to use the https:// protocol instead. If you ever put this site on a server that requires use of the HTTPS protocol, the browser may generate a warning, or may not even load additional resources from HTTP URLs. But the browser will happily load resources from HTTPS URLs if the original page was fetched over HTTP.

Now you need to add a series of style rules, each of which applies a particular set of formatting to elements selected by a CSS selector. The following list describes the formatting you need to apply, but you will have to figure out how to construct your rules so that they are applied to the correct elements. This kind of list is what you would commonly get from a graphic designer, so this is good practice for you. If you've forgotten the property name that controls a particular setting, use the embedded hyperlinks for clues.

General Page Body Formatting

  • All text in the page body should use the font 'Lato' at 18 pixels and 300 weight; if the 'Lato' font is not available for some reason, you should fall back to 'Helvetica Neue', then 'Helvetica', and finally sans-serif
  • The body background image is in the file background.jpg in the img directory (note that the img directory is a peer to the directory in which this CSS file lives, so you need use a relative path that goes up one level and back down into the img directory)
  • The body background image size should be set so that it scales to completely cover the entire page body
  • The background image should also remain fixed with regard to the viewport and not scroll with the page
  • All hyperlinks (both unvisited and visited) should have no underlines, and should have the color #0000AA
  • When a reader hovers over or sets focus to any hyperlink, the color should change to #0000EE

Headings

  • All the headings should use the font 'Raleway' at 300 weight; use the same fall back fonts as you did for the main body font
  • The top-level heading should have a font size of 300% and all margins set to 0
  • The second and third-level headings should use #505050 for their font color
  • The second-level headings should have a font size of 150% and a margin of 0
  • The third-level headings should have a font size of 125%

Main Page Content

  • The main page content should have a Left Margin of 30% so that it is shifted over to the right side of the page by 30% of the current window width; the top, bottom, and right margins should all be 20px
  • The background texture image for that area is in the file ricepaper_v3.png in the img directory
  • The opacity of this area should be set to 0.9 so that is it slightly transparent

Page Header

  • There should be 1em of padding between the edge and the content on all sides
  • The background color should be #FA5A0F
  • The bottom border should be 5px tall in the color #EA4A00
  • The color of the text in the header should be #FFFFFF
  • The tag line ('Feeding Hungry Huskies Since 2013') should be sized at 120% in the color #EEEEEE with all margins set to 0

Restaurant Info Section

  • There should be 1em of padding between the edge and the content on all sides

Menu Section

  • There should be 1em of padding between the edge and the content on all sides
  • The background color for this area should use the following color values: red = 207; green = 205; blue = 140; opacity = 0.5
  • The pizza names should use a font weight of 400. Note that different operating systems may render fonts in slightly different ways. The screen shots were taken on a Mac, so if you are viewing your page on Windows, the 400 font weight may not look quite as dark as it does in the screenshots. The TAs will be viewing these pages on a Mac and looking at your CSS source, so use 400 for the weight even if it looks a bit lighter than the screenshots on Windows.
  • The pizza descriptions should use a font size of 80%, be in italics, and have a 1em margin on the left and bottom

About Us Section

  • There should be 1em of padding between the edge and the content on all sides

Page Footer

  • There should be 1em of padding between the edge and the content on all sides
  • The background color for this area should be #FC892F
  • The top border should be 5px tall in the color #EC791F
  • The text should sized at 80% and aligned on the right side
